# Подбор свечи + Telegram-бот «Волшебный огонь»

Мини-сервис «Modern Mystic Premium»: веб-приложение для подбора свечи по ситуации
и Telegram-бот, который продолжает диалог с клиентом и отправляет заявки админу.

## Структура проекта

```
/candle selection
├── backend/
│   └── server.js          # Express API: /api/send-telegram, /api/track, /api/config
├── bot/
│   └── bot.js             # Telegram-бот (long polling, node-telegram-bot-api)
├── frontend/              # (рекомендуемое место для HTML, см. примечание ниже)
├── candles.json           # База свечей, категорий и текстов сообщений (бот + бэкенд)
├── Подбор свечи.html      # Веб-приложение (главный фронт)
├── index.html             # Редирект на «Подбор свечи.html»
├── leads.json             # Авто-создаётся: лог заявок (НЕ коммитить)
├── events.json            # Авто-создаётся: лог событий (купить/категория/...)
├── package.json
├── .env.example           # Скопируйте в .env и заполните
└── .gitignore
```

> Можно перенести `Подбор свечи.html` и `index.html` в папку `frontend/`. Текущий
> `backend/server.js` отдаёт статику из корня проекта, поэтому работает обе раскладки.

---

## 1. Установка

```bash
# Node.js 18+ обязателен (для встроенного fetch)
npm install
```

## 2. Настройка `.env`

Скопируйте `.env.example` → `.env` и заполните 3 ключевых значения:

```env
TELEGRAM_BOT_TOKEN=123456:ABC-...
TELEGRAM_BOT_USERNAME=volshebnyi_ogon_bot
TELEGRAM_CHAT_ID=123456789
PORT=3000
ALLOWED_ORIGIN=*
```

### 2.1 Получить TOKEN бота
1. Откройте в Telegram бота **@BotFather**.
2. Команда `/newbot` → задайте имя и username (заканчивается на `bot`).
3. BotFather пришлёт строку вида `123456:ABC-DEF...` — это `TELEGRAM_BOT_TOKEN`.
4. Опционально: `/setdescription`, `/setuserpic`, `/setcommands`.
5. Username бота (без `@`) → `TELEGRAM_BOT_USERNAME`.

### 2.2 Получить CHAT_ID (куда приходят заявки)
1. Запустите бота: `npm run bot`.
2. В Telegram нажмите **Start** у вашего бота — в консоли появится строка
   `[/start] chat_id=123456789, ...` Это и есть ваш `TELEGRAM_CHAT_ID`.
3. Альтернатива — бот **@userinfobot**.
4. Чтобы получать уведомления в группу: добавьте бота в группу, отправьте любое
   сообщение, откройте `https://api.telegram.org/bot<TOKEN>/getUpdates` —
   найдите `chat.id` (отрицательное число для групп).

## 3. Запуск

```bash
# Только бэкенд (API + раздача фронта)
npm run server

# Только бот
npm run bot

# Всё вместе (рекомендуется в разработке)
npm run dev
```

Откройте: <http://localhost:3000> — приложение подхватит API на `/api/*`.

> Если открываете `Подбор свечи.html` через VS Code Live Server (порт 5500),
> в начале `<script>` файла измените:
> ```js
> const APP_CONFIG = { API_BASE: "http://localhost:3000/api", BOT_USERNAME: "..." };
> ```
> и в `.env` укажите `ALLOWED_ORIGIN=http://localhost:5500`.

## 4. Как всё работает

```
Клиент в браузере  ──▶  «Подбор свечи.html»
        │
        │ выбирает ситуацию или описывает текстом
        ▼
   Карточка свечи
        │
        ├──▶ кнопка «Купить»                  → ссылка на сайт + /api/track buy_click
        ├──▶ кнопка «Продолжить в Telegram»   → t.me/<bot>?start=<candleId>
        └──▶ форма заявки                     → POST /api/send-telegram
                                                 ├─ leads.json (CRM-минимум)
                                                 ├─ Telegram-уведомление админу
                                                 └─ ответ с botLink → авто-открытие бота
```

В Telegram-боте клиент попадает на:
- готовую карточку свечи (если переход с deep-link),
- меню категорий → карточка → допродажа («с этой свечой берут»),
- мини-диалог «Оставить заявку» — также уходит админу.

Любой выбор категории, нажатие «Купить» (через web) или сообщение в боте
дублируется админу в Telegram.

## 5. Как менять контент

Единый источник правды — **`candles.json`**. И бот, и веб берут свечи из него
(веб — через `GET /api/candles`, бот — напрямую с горячей перезагрузкой).

Каждая свеча описывается так:

```json
{
  "id": "tsarskaya-pechat",
  "name": "Царская печать",
  "category": "Деньги",
  "image": "images/tsarskaya-pechat.png",
  "shortDescription": "Кратко 1–2 предложения.",
  "description": "Развёрнутый текст для карточки и Telegram.",
  "situations": ["Деньги", "Финансы", "Уверенность"],
  "recommendation": "Как зажигать, что сформулировать вслух.",
  "price": 2490,
  "buyLink": "https://ruswedma.ru/products/...",
  "benefits": ["опционально — для блока «что усиливает»"],
  "upsell": ["udacha", "schastlivaya-ruka"]
}
```

- **`situations`** — основное поле для подбора (массив строк).
- **`image`** — относительный путь `images/название.png`. Если файл не
  загрузится в браузере, фронт автоматически покажет премиальную заглушку
  «Modern Mystic Premium».
- Старые поля (`salesPitch`, `usage`, `link`, `situation`) тоже продолжают
  работать — нормализатор `backend/candleNormalize.js` сводит схемы вместе.

### 5.1 Массовое добавление свечей

Шаблон импорта — **`candles-import-template.json`** в корне проекта.
Откройте его, продублируйте объект свечи нужное число раз и заполните поля.

Загрузить можно двумя способами:

1. **Через админ-панель** (рекомендуется).
   В приложении нажмите «Админ-панель» в шапке → блок
   «Массовый импорт свечей из JSON».
   - Выберите файл (или вставьте JSON в текстовое поле).
   - Режим **«Дополнить»** — мерджит по `id` (новые добавляются, существующие
     перезаписываются).
   - Режим **«Заменить весь каталог»** — полностью переписывает массив `candles`
     в `candles.json` (категории и тексты в `messages` не трогаются).
2. **HTTP-запросом**:
   ```bash
   curl -X POST http://localhost:3000/api/admin/import \
     -H "Content-Type: application/json" \
     --data-binary @candles-import-template.json
   ```

После импорта:
- бот автоматически перечитает `candles.json` и сразу увидит новые свечи;
- веб подтянет свежий каталог через `GET /api/candles`;
- подбор свечи (по тегам и по свободному тексту) учитывает все ситуации
  из массива `situations` каждой свечи.

### 5.2 Где хранить картинки

Положите файлы в папку `images/` рядом с `Подбор свечи.html`. В JSON указывайте
путь так: `"image": "images/название.png"`.

Чтобы Telegram-бот мог отдавать локальные файлы как фото свечей, в `.env`
заполните `PUBLIC_BASE_URL` (см. `.env.example`). Для разработки на одной
машине достаточно `http://localhost:3000`. В продакшне — публичный домен
(`https://volshebnyiogon.ru` и т.п.).

### 5.3 Добавление одной свечи без кода

В админ-панели заполните форму «Новая свеча» — данные сразу:
- сохраняются в `localStorage` браузера и попадают в подбор;
- (если backend поднят) отправляются в `POST /api/admin/candle` и
  записываются в `candles.json` — Telegram-бот подхватит автоматически.

## 6. Безопасность

- `TELEGRAM_BOT_TOKEN` и `CHAT_ID` хранятся **только в `.env`**, который добавлен
  в `.gitignore`. Во фронтенде токен не используется — фронт ходит на `/api/*`.
- Все пользовательские данные перед отправкой в Telegram экранируются (`escapeHtml`).
- `cors` ограничен переменной `ALLOWED_ORIGIN` (по умолчанию `*` — для разработки).

## 7. Деплой (кратко)

- **Render / Railway / VPS:** запускаете `npm install && npm run dev`. Откройте
  порт `PORT` для фронта, бэкенд работает там же.
- **Бот:** работает в режиме polling — никакого вебхука и публичного URL не
  требуется. Подойдёт любая VPS / контейнер с постоянным процессом.
- **Webhook вместо polling:** при необходимости можно переключить
  `node-telegram-bot-api` на `webHook` и пробросить `/telegram/webhook` на
  `https://<ваш-домен>/...` — пока этого не нужно.

## 8. Скрипты

| Команда           | Что делает                               |
|-------------------|------------------------------------------|
| `npm run server`  | Запускает Express (API + статика)        |
| `npm run bot`     | Запускает Telegram-бота (polling)        |
| `npm run dev`     | Параллельно сервер + бот                 |

Готово. Заполните `.env`, запустите `npm run dev` — приложение отдаётся на
<http://localhost:3000>, заявки приходят вам в Telegram, бот ведёт клиента к покупке.
